# CMPE314 Software Engineering Labs

## Lab 3 – Use Case Diagrams

### Online Bookstore Use Case Diagram



#### Actors
- **Customer:** Searches and views books.
- **Registered Customer:** Logs in, adds books to cart, and places orders.
- **Administrator:** Manages inventory and orders.
- **Payment Gateway:** Processes payments.
- **Email System:** Sends confirmations and notifications.

#### Use Cases
- **Search Books:** Allows users to search books.
- **View Book Details:** Shows book information.
- **Place Order:** Allows customers to buy books.
- **Process Payment:** Handles payment transactions.
- **Manage Inventory:** Lets admin update books.

---

### Charity Donation Platform Use Case Diagram



#### Actors
- **Donor:** Browses campaigns and makes donations.
- **Non-Profit Organization:** Creates and manages campaigns.
- **Administrator:** Manages users and campaigns.
- **Payment Gateway:** Processes secure donations.
- **Notification System:** Sends updates and alerts.

#### Use Cases
- **Register Account:** Allows users to create accounts.
- **Browse Campaigns:** Lets donors explore campaigns.
- **Search Campaigns:** Helps donors find specific campaigns.
- **Make Donation:** Allows secure donations.
- **Create Campaign:** Lets organizations create campaigns.
- **Manage Campaigns:** Allows admin control.
- **Track Donation Impact:** Lets donors view campaign progress.



## Lab 4 – Sequence Diagrams

### Part 1 – Library Kiosk



#### Description
The sequence diagram models the book return process in a library kiosk. The student returns a book, the system validates it, updates inventory, sends notifications, and confirms success. If the book is overdue, a fine payment process is triggered.

---

### Part 2 – Charity Donation Platform



#### Description
The donor browses campaigns and makes a donation through the platform. The system verifies the user, processes payment through the payment gateway, updates donation records, and sends confirmation notifications. If payment fails, the donation process ends.
